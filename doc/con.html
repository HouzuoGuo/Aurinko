CON - Clojure Object Notation

Back to Index

CON is used everywhere in Aurinko: Document storage, server responses, client requests, etc. CON is natively understood by Clojure programming language.

To serialize an object into CON: (pr-str object_here)
To de-serialize an object: (read-string "CON_string")

The following table lists all the elements that may appear in Aurinko collection documents:

Element Examples
Number 12.345
String "abc123"
Vector ["abc" 123 "def" 45.67]
May contain nested vectors/maps
Keyword :this-is-a-keyword
Case-sensitive, may contain [0-9][A-Z][a-z] or dash '-'
Map {:name "CON" :key2 23.45 :used-by ["Aurinko" "FleetDB"]}
May contain nested maps

A comprehensive example:

{:name "Aurinko"
 :type "Document database engine"
 :depends-on {:Clojure   [1.3 1.4]
              :Leiningen [1.7 2.0]
              :JVM       [1.5 1.6 1.7]
              :OS        ["Unix" "Linux" "Solaris" "Windows"]}}
